Descripcion del proyecto


Frontend


El proyecto consiste en una aplicación web con un frontend minimalista construido con Vite como herramienta de empaquetado. La interfaz presenta un formulario sencillo que permite a los usuarios ingresar datos, los cuales son enviados al backend para su procesamiento.

El backend recibe esta información, la procesa mediante un algoritmo de encriptación, y devuelve como respuesta el código encriptado correspondiente. La arquitectura sigue un enfoque cliente-servidor básico pero efectivo, donde el frontend se encarga de la interacción con el usuario y el backend maneja la lógica de encriptación.

La elección de Vite como bundler asegura un desarrollo ágil y un build optimizado para producción, ofreciendo tiempos de compilación rápidos y un rendimiento eficiente en el lado del cliente.




Backend


Implementa un API REST con dos endpoints para encriptar/desencriptar datos, configurado con CORS restrictivo (solo permite solicitudes desde el frontend del proyecto).

- Tecnologías: Java 17 + Spring Boot 3.x

- Spring Web (MVC)

- Lógica personalizada de encriptación.

- Despliegue en docker.

Características clave:

✅ Seguridad básica: CORS configurado para evitar solicitudes cruzadas no autorizadas.

✅ Escalable: La lógica de encriptación puede reemplazarse (ej: por AES + salting).

✅ Contenedorizado: Listo para despliegue en Docker (si incluyes un Dockerfile).



Configuracion CORS


La restriccion de los cors ya que solo se permite el acceso a este backend desde un unico sitio web.


Wake up


Endpoint para llamar al backend debido a que esta en un servicio gratuito de hosting y este entra en sleep despues de 15 min. de inactividad.


Endpoints del backend


El backend cuenta con dos endpoints uno que encripta palabras y otro que desencripta las palabras encriptadas.
